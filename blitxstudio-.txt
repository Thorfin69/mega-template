  # CLAUDE.md — Blitz Studio Template Repository
                                                                                                                                                                                                                                                
  ## What This System Is                                                                                                                                                                                                                        
                        
  This is the Blitz Studio Template Repository (codename: BLITZ). It is a monorepo of                                                                                                                                                           
  production-ready, client-deployable landing page templates built in React + Vite +                                                                                                                                                            
  Tailwind CSS + Framer Motion. Its purpose is to dramatically compress the time between
  receiving a new client enquiry and delivering a live, polished website.                                                                                                                                                                       
                                                                                                                                                                                                                                                
  The system has two layers:                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                
  1. **The Template Library** — 15–20 standalone landing page templates, each in its own                                                                                                                                                        
     directory, each fully documented, each ready to be cloned and content-swapped for a                                                                                                                                                        
     real client project.                                                               
                                                                                                                                                                                                                                                
  2. **The Platform UI** — a browser-based interface (route `/platform`) where an operator                                                                                                                                                    
     pastes a client brief, selects a template style, and receives a populated prompt that                                                                                                                                                      
     Claude can execute to produce the finished website. This is not an AI API call — it                                                                                                                                                      
     is a structured prompt-builder UI. The output is a Claude prompt, not a deployed site.

  ## The Five Mandatory Documentation Files Per Template                                                                                                                                                                                        
  
  ### 1. `CLAUDE.md` (per template)                                                                                                                                                                                                             
  Every template's CLAUDE.md must contain:                                                                                                                                                                                                    
  - **What this template is** — one paragraph describing the visual style,                                                                                                                                                                      
    emotional register, and ideal use case                                                                                                                                                                                                      
  - **Design law** — a one-line statement of the single most important visual rule                                                                                                                                                              
    (e.g. "Every background is dark. No exceptions.")                                                                                                                                                                                           
  - **Architecture overview** — which components exist, what order they render,                                                                                                                                                                 
    what the router entry point is                                                                                                                                                                                                              
  - **What you must never do** — explicit prohibitions (wrong colors, wrong fonts,                                                                                                                                                              
    breaking the grid, adding sections that don't exist in the template)                                                                                                                                                                        
  - **Content swap protocol** — step-by-step instructions for replacing placeholder                                                                                                                                                           
    content with real client content without breaking the design                                                                                                                                                                                
  - **Animation standard** — the exact Framer Motion pattern used in this template,                                                                                                                                                             
    copied verbatim with the correct values                                                                                                                                                                                                     
  - **Image protocol** — where images come from (Unsplash URLs, local public/,                                                                                                                                                                  
    client-supplied), how they are sized, and what fallback to use                                                                                                                                                                              
                                                                                                                                                                                                                                                
  ### 2. `DESIGN.md` (per template)                                                                                                                                                                                                             
  Must be exhaustive. Every value must be exact. No approximations. Sections:                                                                                                                                                                   
  - Color palette (every hex value with its semantic role)                                                                                                                                                                                      
  - Typography (every font weight, size, line-height, letter-spacing value in use)                                                                                                                                                              
  - Spacing system (section padding, grid gaps, content max-widths)                                                                                                                                                                             
  - Button system (every button variant: primary, ghost, icon — with all classes)                                                                                                                                                               
  - Border radius system (every value in use and where it appears)                                                                                                                                                                              
  - Shadow system                                                                                                                                                                                                                               
  - Animation timing values (duration, easing, stagger intervals, viewport thresholds)                                                                                                                                                          
  - Responsive breakpoints (which breakpoint triggers the layout change and what changes)                                                                                                                                                       
  - Section background pattern (the sequence of bg colors across the page)                                                                                                                                                                      
                                                                                                                                                                                                                                                
  ### 3. `COMPONENTS.md` (per template)                                                                                                                                                                                                         
  A catalog of every component in the template. For each:                                                                                                                                                                                       
  - Component name and file path                                                                                                                                                                                                                
  - Props interface (TypeScript types)
  - Variants (if the component has multiple visual states)                                                                                                                                                                                      
  - Usage example (JSX snippet showing how to instantiate it)                                                                                                                                                                                 
  - Content slots (which text/image props are client-replaceable)                                                                                                                                                                               
  - Design notes (anything non-obvious about the component's visual logic)                                                                                                                                                                      
                                                                                                                                                                                                                                                
  ### 4. `CONTENT.md` (per template)                                                                                                                                                                                                            
  A structured map of every piece of client-replaceable content in the template.                                                                                                                                                                
  Format: a flat list of content slots, each with:                                                                                                                                                                                              
  - Slot ID (e.g. `hero.headline`, `trustbar.stat_1.value`)
  - Current placeholder value                                                                                                                                                                                                                   
  - Content type (text / image URL / number / boolean)                                                                                                                                                                                        
  - Character limit or size constraint                                                                                                                                                                                                          
  - Where it appears in the UI                                                                                                                                                                                                                  
  - Whether it is required or optional                                                                                                                                                                                                          
                                                                                                                                                                                                                                                
  This file is machine-readable. The Platform UI will parse it to generate the                                                                                                                                                                  
  content-swap form for any given template.                                                                                                                                                                                                     
                                                                                                                                                                                                                                                
  ### 5. `PREVIEW.md` (per template)                                                                                                                                                                                                          
  - Route path (e.g. `/template-c`)                                                                                                                                                                                                             
  - Full-page screenshot path (relative: `./preview/full.png`)                                                                                                                                                                                  
  - Section screenshots (hero, mid, footer)
  - Template ID and display name (used in the Platform UI dropdown)                                                                                                                                                                             
  - Industry fits: a tagged list (e.g. `[manufacturing, construction, b2b, industrial]`)                                                                                                                                                        
  - Style keywords (e.g. `[dark, bold, minimal, animated, image-heavy]`)                                                                                                                                                                        
  - Accent color (hex) — used as the swatch in the Platform UI grid                                                                                                                                                                             
  - Complexity rating: Simple / Medium / Complex (how hard to content-swap)                                                                                                                                                                     
  - Estimated content-swap time in hours

  ## The Templates Registry                                                                                                                                                                                                                     
  
  `src/app/platform/templates.registry.ts` is the single source of truth for all                                                                                                                                                                
  templates. It must be updated every time a new template is added. Structure:                                                                                                                                                                
                                                                                                                                                                                                                                                
  ```typescript
  export interface TemplateMetadata {                                                                                                                                                                                                           
    id: string;                    // e.g. "template-c"                                                                                                                                                                                       
    displayName: string;           // e.g. "Bold Industrial"                                                                                                                                                                                    
    route: string;                 // e.g. "/template-c"
    accentColor: string;           // e.g. "#E84D1C"                                                                                                                                                                                            
    bgStyle: "dark" | "light" | "mixed";                                                                                                                                                                                                      
    industries: string[];                                                                                                                                                                                                                       
    keywords: string[];                                                                                                                                                                                                                       
    complexity: "simple" | "medium" | "complex";                                                                                                                                                                                                
    swapHours: number;                                                                                                                                                                                                                          
    previewImage: string;          // URL or relative path to hero screenshot
    claudeMdPath: string;          // relative path to template's CLAUDE.md                                                                                                                                                                     
    contentMdPath: string;         // relative path to template's CONTENT.md                                                                                                                                                                    
    designMdPath: string;          // relative path to template's DESIGN.md                                                                                                                                                                     
  }                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                              
  export const TEMPLATES: TemplateMetadata[] = [                                                                                                                                                                                                
    // one entry per template                                                                                                                                                                                                                 
  ];

  The Platform UI (/platform)                                                                                                                                                                                                                 
                             
  The Platform UI is a single-page tool with three panels:
                                                                                                                                                                                                                                                
  Panel 1 — Template Selector                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                
  - A responsive grid of template cards, one per template in the registry                                                                                                                                                                       
  - Each card shows: hero screenshot, display name, accent color swatch,                                                                                                                                                                      
  industry tags, bg style badge (dark/light), complexity rating                                                                                                                                                                                 
  - Filter bar above the grid: filter by bgStyle, industry, keyword                                                                                                                                                                           
  - Clicking a card selects it and highlights it with a blue ring                                                                                                                                                                               
                                                                                                                                                                                                                                                
  Panel 2 — Client Brief Input                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                
  - A large textarea: "Paste the client enquiry or quotation here"                                                                                                                                                                              
  - Below it: individual structured fields that auto-populate from the brief
  (or can be filled manually):                                                                                                                                                                                                                  
    - Brand name                                                                                                                                                                                                                                
    - Industry / product type                                                                                                                                                                                                                   
    - Tagline (if provided)                                                                                                                                                                                                                     
    - Key selling points (bullet list input)                                                                                                                                                                                                    
    - Target audience                                                                                                                                                                                                                           
    - Reference URLs (if client provided any)                                                                                                                                                                                                   
    - Preferred contact method (phone / form / WhatsApp)                                                                                                                                                                                        
    - Location / city                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                
  Panel 3 — Prompt Output                                                                                                                                                                                                                       
                                                                                                                                                                                                                                              
  - A read-only textarea showing the generated Claude prompt                                                                                                                                                                                    
  - The prompt is constructed by combining:
    a. The selected template's CLAUDE.md (full text)                                                                                                                                                                                            
    b. The selected template's DESIGN.md (full text)                                                                                                                                                                                            
    c. The selected template's CONTENT.md (slot map)
    d. The client brief structured fields injected into a standardized                                                                                                                                                                          
  instruction block at the top                                                                                                                                                                                                                  
  - A "Copy Prompt" button copies to clipboard                                                                                                                                                                                                  
  - A "Open in Claude.ai" button (opens claude.ai/new with the prompt pre-filled                                                                                                                                                                
  if the Claude URL scheme supports it, otherwise just copies)                                                                                                                                                                                  
  - Below the textarea: a preview of the content swap — showing old placeholder                                                                                                                                                                 
  vs new client value for every slot that was auto-detected from the brief

  Global Rules That Apply to ALL Templates                                                                                                                                                                                                    
                                                                                                                                                                                                                                                
  1. Geist is the only font. No Google Fonts. No system fonts in components.
  2. Framer Motion (motion/react) is the only animation library.                                                                                                                                                                                
  No CSS transitions on content. No GSAP. No AOS.                                                                                                                                                                                               
  3. No position: absolute in hand-written components.                                                                                                                                                                                          
  Only Figma-exported or hero background elements may use absolute positioning.                                                                                                                                                                 
  4. Every template must be responsive at exactly two breakpoints:                                                                                                                                                                              
  desktop (≥1024px) and mobile (<1024px). No tablet-specific layouts.                                                                                                                                                                           
  5. Every template must be standalone — its own Navbar and Footer,                                                                                                                                                                             
  no shared layout components between templates.                                                                                                                                                                                                
  6. All images must be real URLs — Unsplash, client-supplied, or public/.                                                                                                                                                                      
  No placeholder boxes. No broken images.                                                                                                                                                                                                       
  7. All scroll animations use viewport={{ once: true, amount: 0.2 }}.                                                                                                                                                                          
  Nothing animates more than once.                                                                                                                                                                                                              
  8. All primary CTAs use the pill shape — rounded-[50px] minimum.                                                                                                                                                                              
  No rectangular buttons as primary actions.                                                                                                                                                                                                    
  9. TypeScript strict mode. No any. No untyped props.                                                                                                                                                                                          
  10. Every template route must be registered in templates.registry.ts                                                                                                                                                                          
  and in the root App.tsx router before the template is considered complete.

  Adding a New Template — Checklist                                                                                                                                                                                                             
                                                                                                                                                                                                                                                
  When Claude is instructed to add a new template, it must complete ALL of the
  following before declaring the template done:                                                                                                                                                                                                 
                                                                                                                                                                                                                                                
  - Create templates/template-x/ directory                                                                                                                                                                                                      
  - Write CLAUDE.md with all required sections                                                                                                                                                                                                  
  - Write DESIGN.md with all design tokens (measured, not approximated)                                                                                                                                                                         
  - Write COMPONENTS.md with full component catalog                                                                                                                                                                                             
  - Write CONTENT.md with complete slot map                                                                                                                                                                                                     
  - Write PREVIEW.md with route, industries, keywords, complexity                                                                                                                                                                               
  - Build index.tsx with all sections (min. 8 sections per template)                                                                                                                                                                          
  - Build all sub-components in components/                                                                                                                                                                                                     
  - Register in templates.registry.ts                                                                                                                                                                                                           
  - Add route to App.tsx                                                                                                                                                                                                                        
  - Run Playwright scroll-screenshot and save to templates/template-x/preview/                                                                                                                                                                  
  - Update PREVIEW.md with actual screenshot paths                                                                                                                                                                                              
                                                                                                                                                                                                                                                
  A template with any checklist item incomplete is not a template — it is a draft.                                                                                                                                                              
  Do not route it. Do not register it. Do not count it.                                                                                                                                                                                         
                                                                                                                                                                                                                                                
  ---                                                                                                                                                                                                                                         
  Platform UI Rules                                                                                                                                                                                                                             
                                                                                                                                                                                                                                              
  - The Platform UI lives at /platform and is always accessible
  - It never modifies template files — it only reads them                                                                                                                                                                                       
  - The generated prompt must always include the full CLAUDE.md and DESIGN.md                                                                                                                                                                   
  of the selected template so Claude has complete context with no follow-up needed                                                                                                                                                              
  - The prompt must end with a clear instruction:                                                                                                                                                                                               
  "Using the template architecture and design system above, replace all content                                                                                                                                                                 
  slots with the client brief below and output the complete, deployment-ready                                                                                                                                                                   
  React component files."                                                                                                                                                                                                                       
  - The Platform UI must work offline — no external API calls
